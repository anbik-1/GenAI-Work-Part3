# short-infra-deploy-summary.md

A personal reference explaining how the AnyCompanyRead infrastructure was created in AWS,
and where the frontend and backend actually live.

---

## 1. How Was the Infrastructure Created in AWS?

**One command: `npx cdk deploy`**

The tool behind it is **AWS CDK (Cloud Development Kit)** — it lets you describe AWS
infrastructure as TypeScript code, then automatically provisions everything in AWS.

### The Chain

```
TypeScript code          CDK translates         CloudFormation
in anycompanyread-    →  it to a JSON       →   creates all real
stack.ts                 template               AWS resources
(WHAT you want)          (CloudFormation)       (101 resources)
```

### Step by Step

**Step 1 — Infrastructure described as TypeScript**

Every AWS resource is a TypeScript object in:
`anycompanyread/packages/infrastructure/lib/anycompanyread-stack.ts`

```typescript
// Creates a Cognito User Pool
const userPool = new cognito.UserPool(this, 'UserPool', {
  userPoolName: 'AnyCompanyRead-UserPool',
  selfSignUpEnabled: true,
  signInAliases: { email: true },
});

// Creates a DynamoDB table
const booksTable = new dynamodb.Table(this, 'BooksTable', {
  tableName: 'AnyCompanyRead-Books',
  partitionKey: { name: 'bookId', type: dynamodb.AttributeType.STRING },
  billingMode: dynamodb.BillingMode.PAY_PER_REQUEST,
});

// Bundles the Lambda handler with esbuild and deploys it
const booksFunction = new NodejsFunction(this, 'BooksFunction', {
  entry: path.join(backendSrc, 'handlers/books/index.ts'),
  runtime: lambda.Runtime.NODEJS_20_X,
  environment: commonEnv,
});
```

**Step 2 — `cdk synth` translates TypeScript → CloudFormation template (JSON)**

CDK compiles the code into a giant JSON template describing every resource and
relationship. Lambda code is bundled by esbuild at this step.

**Step 3 — `cdk deploy` sends the template to AWS CloudFormation**

CloudFormation creates all 101 resources in the correct order (e.g. Cognito must
exist before the API Gateway authorizer that references it).

Deployment output looked like:
```
AnyCompanyReadStack |   1/101 | CREATE_IN_PROGRESS | AWS::IAM::Role ...
AnyCompanyReadStack |  50/101 | CREATE_COMPLETE    | AWS::Lambda::Function ...
AnyCompanyReadStack | 101/101 | CREATE_COMPLETE    | AWS::CloudFormation::Stack
```

**Step 4 — Credentials came from the EC2 instance profile**

No AWS username or password was needed. The EC2 machine running this IDE has an
**IAM instance role** (`workshop-stack-IdeAdminRole`) pre-attached. CDK used that
role automatically. This is how AWS services authenticate to each other — no keys
in files, no passwords.

### What CDK Does Automatically (the "magic")

CDK handles all the **wiring** between services:
- Gave each Lambda the exact IAM permissions it needs (Books Lambda → read Books table, etc.)
- Connected API Gateway routes to the correct Lambda functions
- Configured CORS on all API routes
- Wired the Cognito authorizer to the protected routes
- Set all Lambda environment variables pointing to the correct table names and User Pool

You just described WHAT you want. CDK figured out HOW to connect it all.

### Infrastructure as Code — Project Structure

```
packages/infrastructure/
├── bin/
│   └── app.ts                    ← CDK entry point (instantiates the stack)
└── lib/
    └── anycompanyread-stack.ts   ← ALL AWS resources defined here as TypeScript
```

Run order:
```bash
npx cdk bootstrap   # one-time: creates CDK support resources in the account
npx cdk synth       # compile TypeScript → CloudFormation template
npx cdk deploy      # send template to AWS, create all resources
npx cdk destroy     # tear everything down (cleans up all resources)
```

## 2. Where Is the Frontend (FE)?

### In the Codebase

```
anycompanyread/packages/frontend/
├── src/
│   ├── pages/          ← 7 pages: Home, Books, BookDetail, Login, Signup, Cart, Orders
│   ├── components/     ← shadcn/ui base components, Layout (Navbar/Footer), BookCard, BookGrid
│   ├── contexts/       ← AuthContext (login/signup/logout), CartContext (cart state)
│   ├── lib/
│   │   ├── api.ts      ← HTTP client (fetch wrapper with auth token injection)
│   │   └── utils.ts    ← cn() Tailwind class utility
│   ├── App.tsx         ← React Router routes + all providers
│   └── main.tsx        ← Entry point
└── dist/               ← Production build output (generated by `npm run build`)
```

### Live in AWS

```
npm run build:frontend
        ↓
packages/frontend/dist/
  ├── index.html          (0.58 KB)
  └── assets/
      ├── index.css       (32.7 KB — Tailwind styles)
      └── index.js        (348.8 KB — all React code bundled)
        ↓
aws s3 sync packages/frontend/dist/ s3://anycompanyread-frontend-654654306837-us-east-1/
        ↓
CloudFront CDN (d3dzjyyemejat.cloudfront.net)
  - Serves files globally over HTTPS
  - SPA fallback: 403/404 errors → index.html (so React Router works)
  - No server running — just static files served from S3 via CloudFront
```

**Key point:** The frontend is just static files. There is no web server. CloudFront
serves HTML/JS/CSS from S3 directly to the user's browser. React then runs in the
browser and calls the backend API.

**Live URL:** https://d3dzjyyemejat.cloudfront.net

```
│   ├── cart/

│   │   ├── get-cart.ts                 ← DynamoDB Query by userId
│   │   ├── add-to-cart.ts              ← DynamoDB PutItem (with book denormalization)

│   └── orders/

│       ├── checkout.ts                 ← Read cart → create order → clear cart
│       ├── list-orders.ts              ← DynamoDB Query by userId
│       └── get-order.ts                ← DynamoDB GetItem + Query order items

    ├── dynamodb.ts   ← Shared DynamoDB Document Client
    ├── response.ts   ← Lambda response helpers (success, error, badRequest, etc.)

```

### Live in AWS

```
Each handler  →  esbuild bundles TypeScript  →  Lambda function (zip, ~5-7 KB)
                 (shared package included,        runs only when called
                  AWS SDK excluded — provided      scales to zero when idle)
                  by Lambda runtime)

              (routes each URL path to the right Lambda)
                        ↓
              DynamoDB (4 tables)
              (persists all data)
```

**Live API URL:** https://94c5hkt4l7.execute-api.us-east-1.amazonaws.com/prod/

## 4. How Frontend and Backend Connect

```
User's Browser
      │
      │  Opens https://d3dzjyyemejat.cloudfront.net
      ▼
 CloudFront ──→ serves React app (HTML/JS/CSS) from S3
      │
      │  React app runs in the browser, user logs in / browses books
      │  React makes API calls to the backend using fetch()
      ▼
 API Gateway
 https://94c5hkt4l7.execute-api.us-east-1.amazonaws.com/prod/
      │
      ├── /auth/*    ──→ AnyCompanyRead-Auth Lambda    ──→ Amazon Cognito
      │                  (handles signup, login)             (user directory)
      │
      ├── /books/*   ──→ AnyCompanyRead-Books Lambda   ──→ DynamoDB Books
      │                  (catalog, search, filter)
      │
      ├── /cart/*    ──→ AnyCompanyRead-Cart Lambda    ──→ DynamoDB Carts
      │                  (add, update, remove items)
      │
      └── /orders/*  ──→ AnyCompanyRead-Orders Lambda  ──→ DynamoDB Orders
         /checkout       (checkout, order history)          + OrderItems
                                                            + Carts (cleared)
```

**There is no traditional server.** The backend is 4 Lambda functions that wake up
on demand when API Gateway receives a request, execute in milliseconds, and go back
to sleep. This is the serverless model — you pay only for the time they actually run.

## 6. Key Commands Reference

```bash
# Build everything
cd anycompanyread
npm run build:shared    # compile shared types first
npm run build:backend   # compile Lambda handlers
npm run build:frontend  # Vite production bundle

# Deploy infrastructure
cd packages/infrastructure
npx cdk bootstrap       # one-time account setup
npx cdk deploy          # create/update all AWS resources

# Deploy frontend (after cdk deploy gives you bucket name)
aws s3 sync packages/frontend/dist/ s3://<FrontendBucketName>/ --delete

# Seed the database
cd anycompanyread
npx ts-node scripts/seed-data.ts

# Tear everything down
cd packages/infrastructure
npx cdk destroy
```
